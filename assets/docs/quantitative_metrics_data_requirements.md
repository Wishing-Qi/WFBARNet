# 羽毛球量化指标与数据需求分析

本文档整理 WFBARNet 当前项目已有数据能够支持哪些羽毛球分析，以及各项指标需要哪些输入数据。文档重点面向项目实现、数据页设计、答辩说明和后续功能规划。

## 1. 结论概览

WFBARNet 当前最适合落地的量化路线是：

```text
视频帧
  -> 羽毛球轨迹
  -> 人体姿态
  -> 球场映射
  -> 轨迹事件
  -> 球员场地投影
  -> 回合统计与可视化
```

因此，当前项目已经能够较好支持以下分析：

| 分析方向 | 当前支持程度 | 说明 |
| --- | --- | --- |
| 回合节奏分析 | 支持 | 回合时长、击球次数、平均击球间隔、落点次数、出画次数 |
| 羽毛球轨迹事件分析 | 支持 | `hit`、`landing`、`out_of_frame` 事件及触发规则 |
| 球员移动负荷分析 | 支持 | 跑动距离、平均速度、最大速度、启动、停止、高强度移动 |
| 球员站位与空间分析 | 支持 | 平均站位深度、前后移动比例、左右移动比例、标准场地投影 |
| 击球区域分析 | 部分支持 | 可统计前场/中场/后场击球；击球归属依赖 BST 或球员位置推断 |
| 击球动作类型分析 | 可选支持 | 依赖本地 BST 击球动作识别权重 |
| 数据可靠性分析 | 支持 | 球点可见率、姿态有效率、球场有效率、平均球置信度 |
| 模型调试与误检排查 | 支持 | 逐帧 JSONL、轨迹调试 CSV、事件规则、滤波动作 |
| 比分与胜负分析 | 暂不支持 | 缺少比分、得失分、发球权、胜负标签 |
| 制胜分/非受迫失误分析 | 暂不支持 | 需要人工标注或更高层语义模型 |
| 真实球速/杀球速度 | 暂不可靠 | 单目视频下羽毛球不总在球场平面，缺少 3D 或雷达数据 |
| 生理负荷与恢复分析 | 暂不支持 | 需要心率、HRV、sRPE、睡眠、酸痛等外部数据 |

## 2. 当前项目已有数据

### 2.1 视频与时间数据

| 数据 | 来源 | 用途 |
| --- | --- | --- |
| `frame_id` | 视频读取或摄像头循环 | 对齐逐帧结果、事件和调试记录 |
| `timestamp_ms` | 视频 FPS 或实时计时器 | 计算回合时长、速度、事件间隔 |
| 原始视频帧 | 本地视频或摄像头 | 模型推理、可视化叠加、调试回放 |
| FPS | 视频元数据或默认配置 | 时间换算、轨迹窗口、事件检测 |

### 2.2 羽毛球轨迹数据

| 数据 | 来源 | 用途 |
| --- | --- | --- |
| TrackNet 热力图 | `TrackBranch` | 解码候选球点 |
| 候选球点列表 | `infer_candidate_results(...)` | 多候选选择、误检分析 |
| `ball_xy` | 轨迹滤波输出 | 轨迹显示、事件检测、回合统计 |
| `visible` | 轨迹滤波输出 | 判断球点是否可用 |
| `score` | TrackNet/滤波输出 | 数据可靠性、事件过滤 |
| `heatmap_shape` | 轨迹结果 | 热力图来源与缩放排查 |

### 2.3 轨迹事件数据

| 数据 | 来源 | 用途 |
| --- | --- | --- |
| `hit_event` | `RealtimeTrajectoryEventDetector` | 击球次数、击球时间、BST 输入 |
| `landing_event` | `RealtimeTrajectoryEventDetector` | 落点次数、回合结束 |
| `out_of_frame` | `RealtimeTrajectoryEventDetector` | 跟踪丢失/出画计数 |
| `rule` | 轨迹事件 | 解释事件触发原因 |
| `confidence` | 轨迹事件 | 事件可信度 |
| `features` | 轨迹事件 | 速度、反转、局部特征调试 |

### 2.4 姿态与球员数据

| 数据 | 来源 | 用途 |
| --- | --- | --- |
| 人体框 `bbox` | YOLO Pose / MMPose | 球员检测、脚下点估计 |
| 人体关键点 `keypoints` | YOLO Pose / MMPose | 骨架绘制、动作识别输入 |
| 关键点置信度 | 姿态模型 | 姿态有效性判断 |
| 稳定球员身份 | `CourtPoseTargetTracker` | 区分上方/下方球员 |
| 球员脚下点 | 姿态后处理 | 投影到标准球场 |

### 2.5 球场与空间映射数据

| 数据 | 来源 | 用途 |
| --- | --- | --- |
| 球场线/角点 | 球场检测模块 | 估计球场几何 |
| `image_to_court_h` | 单应性矩阵 | 图像坐标转场地坐标 |
| `court_valid` | 球场检测结果 | 判断空间指标是否可信 |
| 标准球场坐标 | 投影结果 | 跑动距离、站位、热力图、击球区域 |

标准羽毛球场尺寸在项目中按以下常量处理：

```text
COURT_WIDTH_CM = 610
COURT_LENGTH_CM = 1340
HALF_COURT_LENGTH_CM = 670
```

### 2.6 BST 击球动作识别数据

| 数据 | 来源 | 用途 |
| --- | --- | --- |
| `pred_name` | BST 模型 | 击球动作类别 |
| `pred_display_name` | BST 模型 | UI 展示名称 |
| `confidence` | BST 模型 | 动作识别可信度 |

该模块是可选能力。没有 BST 权重时，系统仍可完成轨迹、姿态、球场、事件和回合统计。

| 数据 | 位置 | 用途 |

## 3. 当前可支持的分析

### 3.1 回合节奏分析

当前项目可以把一个视频或分析片段作为一个回合统计单元，输出回合状态与节奏指标。

| 指标 | 所需数据 | 当前来源 | 支持程度 |
| --- | --- | --- | --- |
| 回合开始时间 | 连续可见球点或第一条 `hit` | 轨迹结果、`hit_event` | 支持 |
| 回合结束时间 | 可信 `landing` | `landing_event` | 支持 |
| 回合时长 | 开始时间、结束时间 | `RallyStatsAccumulator` | 支持 |
| 回合状态 | 开始/结束是否存在 | `summary().rally_state` | 支持 |
| 击球次数 | `hit_event` 数量 | 轨迹事件/BST 合并 | 支持 |
| 平均击球间隔 | 多个击球时间戳 | `summary().avg_hit_interval_ms` | 支持 |
| 落点次数 | `landing_event` 数量 | 轨迹事件 | 支持 |
| 出画次数 | `out_of_frame` 数量 | 轨迹事件 | 支持 |

当前限制：

- 尚未实现完整多回合自动切分。
- 一个输入视频如果包含多个回合，当前更适合先切成片段再分析。
- 回合结束依赖 `landing` 事件质量；误检会提前结束，漏检会保持回合中。

### 3.2 羽毛球轨迹与事件分析

当前项目能够从 TrackNetV3 风格轨迹结果中提取可解释事件。

| 分析 | 所需数据 | 当前来源 | 支持程度 |
| --- | --- | --- | --- |
| 轨迹可视化 | `ball_xy`, `visible`, `score` | 滤波后 `TrackResult` | 支持 |
| 击球点检测 | 滤波轨迹、速度方向变化 | `RealtimeTrajectoryEventDetector` | 支持 |
| 落点检测 | 速度突变、低速尾段、轨迹结束 | `RealtimeTrajectoryEventDetector` | 支持 |
| 出画/丢失检测 | 可见性变化、边缘位置 | `RealtimeTrajectoryEventDetector` | 支持 |
| 事件可信度分析 | `confidence`, `rule`, `features` | 轨迹事件 | 支持 |
| 轨迹误检排查 | 候选点、滤波动作、分数 | `track_debug.csv` | 支持 |

当前主击球只接受反转类规则：

```text
vy_reversal
vx_reversal
```

以下规则只作为辅助证据或其它事件来源，不单独生成主击球点：

```text
acceleration_peak
speed_local_max
y_local_max
```

### 3.3 球员移动负荷分析

球员移动指标依赖姿态点投影到标准球场平面后的连续坐标。

| 指标 | 所需数据 | 当前来源 | 支持程度 |
| --- | --- | --- | --- |
| 累计跑动距离 | 连续球员场地坐标、时间戳 | `player_points` + `image_to_court_h` | 支持 |
| 平均速度 | 跑动距离、有效跟踪时长 | `RallyStatsAccumulator` | 支持 |
| 最大速度 | 相邻有效点速度 | `RallyStatsAccumulator` | 支持 |
| 启动次数 | 前后速度状态变化 | `start_speed_mps` 阈值 | 支持 |
| 停止/急停次数 | 速度从移动降到低速 | `stop_speed_mps` 阈值 | 支持 |
| 高强度移动次数 | 速度超过高强度阈值 | `high_intensity_speed_mps` | 支持 |
| 最大连续移动距离 | 连续移动段累计距离 | `RallyStatsAccumulator` | 支持 |
| 运动强度分 | 距离、时长、高强度次数 | `motion_intensity_score` | 支持 |

当前默认阈值：

```text
min_step_cm = 2.0
max_step_cm = 180.0
start_speed_mps = 1.20
stop_speed_mps = 0.35
high_intensity_speed_mps = 3.00
```

其中：

- 小于 `2 cm` 的步长视为姿态抖动，不累计。
- 大于 `180 cm` 的步长视为跟踪跳变，重置上一点。
- 达到 `1.20 m/s` 记为进入移动状态。
- 降到 `0.35 m/s` 附近记为退出移动状态。
- 达到 `3.00 m/s` 记为高强度移动。

### 3.4 空间站位与热力图分析

空间分析依赖球场检测有效，并且人体位置能够稳定投影到标准场地。

| 指标 | 所需数据 | 当前来源 | 支持程度 |
| --- | --- | --- | --- |
| 球员标准场地位置 | 姿态脚下点、单应性矩阵 | `project_player_points_to_court(...)` | 支持 |
| 站位热力图 | 连续球员场地坐标 | UI/热力图渲染 | 支持 |
| 平均站位深度 | 球员场地 y 坐标 | `avg_stance_depth_cm` | 支持 |
| 前后移动比例 | 连续场地坐标的 y 方向变化 | `front_back_movement_ratio` | 支持 |
| 左右移动比例 | 连续场地坐标的 x 方向变化 | `left_right_movement_ratio` | 支持 |
| 球点场地投影 | 球点图像坐标、单应性矩阵 | 可用于事件空间分析 | 部分支持 |

注意：

- 球员脚下点在球场平面上，适合用单应性矩阵投影。
- 羽毛球在空中时不一定位于球场平面，用单应性直接投影球点会有高度误差，因此更适合做落点、近地事件或粗略区域参考，不适合直接计算真实 3D 球速。

### 3.5 击球区域与动作类型分析

击球区域统计需要知道“谁在什么时候击球”，以及击球时球员在场地中的位置。

| 指标 | 所需数据 | 当前来源 | 支持程度 |
| --- | --- | --- | --- |
| 球员击球次数 | 击球事件、击球归属 | BST 前缀或球员位置推断 | 部分支持 |
| 前场击球次数 | 击球时球员场地位置 | `zone_hits.front` | 支持，依赖归属 |
| 中场击球次数 | 击球时球员场地位置 | `zone_hits.mid` | 支持，依赖归属 |
| 后场击球次数 | 击球时球员场地位置 | `zone_hits.back` | 支持，依赖归属 |
| 击球动作分布 | BST 动作类别 | `stroke_distribution` | 可选支持 |
| 击球置信度均值 | 事件/BST 置信度 | `hit_confidence_avg` | 支持 |
| 被动击球次数 | 击球时球员速度 | `passive_hit_count` | 支持，依赖归属 |

击球区域按照每个球员半场深度三等分：

```text
标准场地长度 = 1340 cm
半场长度 = 670 cm
半场三等分 = 223.3 cm
靠近网前为 front
中间区域为 mid
远离网前为 back
```

被动击球当前使用速度阈值：

```text
passive_hit_speed_mps = 0.75
```

击球时球员速度低于该阈值，则计入 `passive_hit_count`。

### 3.6 数据可靠性分析

所有量化结果都应该结合数据可靠性一起展示，否则容易把低质量视频中的偶然结果误解为真实表现。

| 指标 | 所需数据 | 当前来源 | 支持程度 |
| --- | --- | --- | --- |
| 球点可见率 | 可见球点帧数、总帧数 | `ball_visible_rate` | 支持 |
| 姿态有效率 | 有效姿态帧数、总帧数 | `pose_valid_rate` | 支持 |
| 球场有效率 | 有效球场帧数、总帧数 | `court_valid_rate` | 支持 |
| 平均球置信度 | 每帧球点分数 | `avg_ball_confidence` | 支持 |
| 事件置信度 | 事件置信度 | `hit_confidence_avg` | 支持 |

推荐解释规则：

| 情况 | 建议解释 |
| --- | --- |
| 球点可见率低 | 击球次数、落点、轨迹事件可信度下降 |
| 姿态有效率低 | 球员距离、速度、站位和击球归属可信度下降 |
| 球场有效率低 | 所有基于场地坐标的指标都应谨慎解释 |
| 平均球置信度低 | 可能存在漏检、误检、运动模糊或模型不适配 |

## 4. 指标数据需求总表

### 4.1 回合与事件指标

| 指标 | 计算方式 | 必需数据 | 当前数据字段 | 支持程度 |
| --- | --- | --- | --- | --- |
| 回合开始时间 | 稳定球飞行起点或第一条 `hit` 时间 | `ball_visible`, `ball_xy`, `ball_score`, `hit_event` | `rally_start_ms` | 支持 |
| 回合结束时间 | 第一条可信 `landing` 时间 | `landing_event.timestamp_ms` | `rally_end_ms` | 支持 |
| 回合时长 | `(end - start) / 1000` | 开始/结束时间 | `rally_duration_s` | 支持 |
| 回合状态 | 是否已有开始/结束 | 开始/结束时间 | `rally_state` | 支持 |
| 处理帧数 | 回合内帧数累计 | `frame_id` | `frame_count` | 支持 |
| 击球次数 | `count(hit_event)` | `hit_event` | `rally_hit_count` | 支持 |
| 平均击球间隔 | 相邻 hit 时间差均值 | 多个 hit 时间戳 | `avg_hit_interval_ms` | 支持 |
| 落点次数 | `count(landing_event)` | `landing_event` | `landing_count` | 支持 |
| 出画次数 | `count(out_of_frame)` | `out_of_frame` | `out_of_frame_count` | 支持 |

### 4.2 球员移动指标

| 指标 | 计算方式 | 必需数据 | 当前数据字段 | 支持程度 |
| --- | --- | --- | --- | --- |
| 累计跑动距离 | `sum(valid_step_cm) / 100` | 球员连续场地坐标 | `players.*.distance_m` | 支持 |
| 平均速度 | `distance_m / tracked_seconds` | 距离、有效跟踪时间 | `players.*.avg_speed_mps` | 支持 |
| 最大速度 | `max(step_m / dt_s)` | 相邻点距离、时间差 | `players.*.max_speed_mps` | 支持 |
| 启动次数 | 低速进入移动阈值 | 逐步速度 | `players.*.start_count` | 支持 |
| 停止次数 | 移动状态降至低速 | 逐步速度 | `players.*.stop_count` | 支持 |
| 高强度移动次数 | 速度超过高强度阈值 | 逐步速度 | `players.*.high_intensity_count` | 支持 |
| 最大连续移动距离 | 连续移动段距离最大值 | 逐步速度、距离 | `players.*.max_continuous_m` | 支持 |
| 运动强度分 | 距离/时长 + 高强度次数 | 双方距离、回合时长 | `motion_intensity_score` | 支持 |

### 4.3 空间与站位指标

| 指标 | 计算方式 | 必需数据 | 当前数据字段 | 支持程度 |
| --- | --- | --- | --- | --- |
| 平均站位深度 | 场地 y 深度均值 | 球员场地坐标 | `players.*.avg_stance_depth_cm` | 支持 |
| 前后移动比例 | 向网前移动距离 / 前后总移动距离 | 连续场地坐标 | `players.*.front_back_movement_ratio` | 支持 |
| 左右移动比例 | 向左移动距离 / 左右总移动距离 | 连续场地坐标 | `players.*.left_right_movement_ratio` | 支持 |
| 站位热力图 | 场地坐标二维分布 | 球员场地坐标序列 | UI 热力图数据 | 支持 |
| 击球区域 | 击球时球员半场深度三等分 | hit 时间、球员场地坐标 | `zone_hits` | 部分支持 |

### 4.4 击球动作与质量指标

| 指标 | 计算方式 | 必需数据 | 当前数据字段 | 支持程度 |
| --- | --- | --- | --- | --- |
| 击球动作分布 | 按动作类别计数 | BST 预测类别 | `stroke_distribution` | 可选支持 |
| 击球置信度均值 | 事件/BST 置信度均值 | `confidence` | `hit_confidence_avg` | 支持 |
| 球员击球次数 | 按球员归属统计 hit | 击球球员侧 | `players.*.hit_count` | 部分支持 |
| 前中后场击球数 | 按球员半场区域统计 | hit 时间、球员位置、球员归属 | `players.*.zone_hits` | 部分支持 |
| 被动击球次数 | 击球时速度低于阈值 | hit 时间、球员速度 | `players.*.passive_hit_count` | 部分支持 |
| Top-K 动作候选 | BST Top-K 输出 | BST 模型输出 | UI payload | 可选支持 |

“部分支持”的主要原因是击球归属并非总能从轨迹事件直接确定。推荐优先级：

1. 使用 BST 类别名中的 `Top_*` / `Bottom_*` 前缀。
2. 使用击球时距离球点最近的球员场地投影。
3. 若没有可靠归属，则记为 unknown，不强行分配给某名球员。

### 4.5 数据质量与调试指标

| 指标 | 计算方式 | 必需数据 | 当前数据字段 | 支持程度 |
| --- | --- | --- | --- | --- |
| 球点可见率 | 可见球帧数 / 总帧数 | `visible` | `data_reliability.ball_visible_rate` | 支持 |
| 姿态有效率 | 有效姿态帧数 / 总帧数 | pose 结果 | `data_reliability.pose_valid_rate` | 支持 |
| 球场有效率 | 有效球场帧数 / 总帧数 | court 结果 | `data_reliability.court_valid_rate` | 支持 |
| 平均球置信度 | 球点分数均值 | `ball.score` | `data_reliability.avg_ball_confidence` | 支持 |
| relock 次数 | 轨迹重锁定动作计数 | `track_debug.csv` | `action/reason` | 支持 |
| inpaint 次数 | 线性补点计数 | `track_debug.csv` | `inpaint_mask` | 支持 |
| 低置信候选比例 | 候选分数统计 | `track_debug.csv` | `input_score/output_score` | 支持 |

## 5. 当前不支持或不建议直接使用的指标

| 指标 | 当前问题 | 需要补充的数据 |
| --- | --- | --- |
| 真实羽毛球速度 | 单目平面投影不能可靠表示空中 3D 速度 | 高帧率视频、3D 重建、双目/多机位、雷达 |
| 杀球速度 | 击球瞬间速度高、模糊大，且有高度变化 | 雷达、高帧率视频、3D 轨迹 |
| 制胜分率 | 系统不知道该球是否直接得分 | 得分结果、末拍标注、比分信息 |
| 非受迫失误率 | 需要判断主动/被动和失误性质 | 人工标注或高级语义模型 |
| 发球效果 | 当前没有稳定发球识别和得分结果 | 发球事件、接发结果、比分 |
| 接发球得分率 | 缺少接发球与得分关联 | 发接发标注、比分 |
| 工作休息比 | 缺少多回合自动切分和间歇识别 | 多回合边界、休息段检测 |
| 心率/HRV/sRPE | 视频无法提供生理负荷 | 心率带、问卷、恢复记录 |
| 睡眠/酸痛/疲劳 | 视频无法提供恢复状态 | 睡眠记录、主观量表 |
| 关节力矩/地面反作用力 | 姿态关键点不足以直接推算真实力学负荷 | 测力台、IMU、足底压力、标定模型 |
| 多回合完整比赛统计 | 当前不自动创建下一回合 | 回合切分算法、比分牌 OCR、长静止检测、人工切片 |

## 7. 指标解释建议

为了避免误读，展示指标时建议同时显示数据质量提示：

```text
如果 court_valid_rate 较低：
  不强调跑动距离、站位深度、前中后场击球分布。

如果 pose_valid_rate 较低：
  不强调球员移动速度、启动/停止次数、被动击球。

如果 ball_visible_rate 或 avg_ball_confidence 较低：
  不强调击球次数、落点次数、平均击球间隔。

如果没有 BST 权重：
  不展示击球动作类型，只展示轨迹事件层面的击球次数。
```

## 8. 后续数据扩展方向

如果要从“视频复盘工具”升级为更完整的羽毛球运动员分析系统，建议按优先级补充以下数据：

1. 多回合边界数据：人工切片、长静止检测、发球检测、比分牌 OCR。
2. 得失分标签：每回合胜负、末拍结果、制胜分、非受迫失误。
3. 击球归属数据：更稳定的击球球员判定、双方交替约束。
4. 发接发数据：发球类型、接发位置、前三拍结果。
5. 外部生理数据：心率、HRV、sRPE、睡眠、酸痛。
6. 高速或多机位数据：真实球速、杀球速度、3D 轨迹。
7. 动作力学数据：IMU、足底压力、测力台或无标记动作分析模型。

## 9. 一句话总结

当前 WFBARNet 已经具备“球轨迹 + 姿态 + 球场映射 + 事件检测 + 回合统计”的完整视频量化基础，最适合分析回合节奏、击球事件、球员移动、站位空间和数据可靠性。比分胜负、制胜分、非受迫失误、真实球速、生理负荷和动作力学等指标不应直接从当前数据中硬推，需要额外标注、传感器或更高层模型支持。
